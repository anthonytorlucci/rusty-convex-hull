# A Rusty Convex Hull in 2D

> [!CAUTION]
> Incomplete... the example runs, but the code needs more documentation and the README needs to be more complete and descriptive.

Imagine you have a scattered set of data points, like cities on a map, stars in the sky, or even measurements from an experiment. You have been given the task to find the smallest convex shape that completely encloses all of these points. This encompassing boundary is known as the *convex hull*.

Implementing a robust and efficient solution for calculating the convex hull is a cornerstone skill for any programmer dealing with geometric algorithms. This article will delve into the details of implementing the Graham Scan algorithm, exploring the core steps and computational considerations necessary to transform a cloud of points into its defining geometric boundary.

> [!NOTE]
> We only consider a collection of 2D points on a Cartesian grid for this example. Higher dimensional convex hull solutions exist, but this implementation is strictly 2D.

## Problem Statement
Given a list of 2D points (each represented as a tuple `(x,y)` where `x` and `y` are integers or floats), compute the **convex hull** of these points. Your task is to return the vertices of this polygon as a subset of the input points, ordered either clockwise (CW) or counter-clockwise (CCW) to form a closed loop.

For example, an input set might look like
```rust
let input_set = [(-1.0, -10.0), (0.0, 0.0), (-3.0, 5.0), (6.0, 1.0)];
```

## Solution

### Convex Set and Convex Hull

A set of points $C$ is called **convex** if for any two points $p$ and $q$ in $C$, the entire segment is contained in $C$. The **convex hull** of a set of points is the smallest convex set containing $S$.^[1](#ref-item-1) for more information on convex sets, see [2](#ref-item-2).

We can think of a convex hull like a rubber band wrapped around all the points without any bends (all angles must be less than 180 degrees). This means that any colinear points should be excluded and only the endpoints of the line segments are retained.

### Graham Scan Algorithm

The Graham Scan is arguably the most famous textbook convex hull algorithm, primarily because it clearly demonstrates the power of sorting combined with geometric checks.

* **Description:**
  1. Find the point with the minimum y-coordinate (the **anchor point** $P\_0$).
  2. **Sort** all other points based on the **angle** they make with the anchor point and the positive x-axis (or $P\_0$'s horizontal line). This is the key step and can be done efficiently using the cross-product to avoid explicit angle calculations.
  3. Iterate through the sorted points using a stack, similar to the Monotone Chain. For each new point, check if the last three points (second-to-last, last, new) form a **left turn**.
     * If it's a **right turn** (or non-left turn), the last point on the stack must be inside the hull and is popped.
     * If it's a **left turn**, the new point is pushed onto the stack.
* **Ease of Implementation:** Most challenging. While the stack-based iteration is similar to Monotone Chain, the **angular sorting** is the difficult part. It requires robust handling of collinear points, measuring angles implicitly using cross-products, and selecting a correct starting point ($P\_0$). If done incorrectly, the sorted order can be wrong, leading to an incorrect hull.

**Why is this algorithm optimal?**

1. Time Complexity: The Graham scan algorithm runs in O(n log n) time, where n is the number of points. This is optimal because sorting (which takes O(n log n) time) is a lower bound for the convex hull problem.
2. Space Complexity: It uses O(n) additional space, which is optimal as we need to store the result.
3. Correctness: It correctly finds the convex hull, which by definition is the smallest convex polygon that contains all the points.
4. Minimum Perimeter: The convex hull has the minimum perimeter among all polygons that encompass all points. Any polygon with a smaller perimeter would necessarily exclude some points.
